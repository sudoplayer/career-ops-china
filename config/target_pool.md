## Tier C — 创业公司（备选，15-30%）

**8 月底前不主动投。** 只有 Tier A/B 都失败 + 4 个月没 offer 时再考虑。

### C1. 二线 AI Agent 创业

候选公司：蝴蝶效应（Manus，2026-04-27 已被 Meta 收购，国内通道关闭）、智谱 Agent 团队、Coze 生态 ISV、各种垂直 Agent（法律 / 医疗 / 电商）创业。

**真实风险**：
- 工作强度高（基本 996）
- A 轮失败率 60%+；期权大概率作废
- gap 1 年的心力账上，这个选项要慎重

**投递建议**：
- 优先选已经有真实付费收入的公司
- 不选刚拿 A 轮还在烧钱的

### C2. 出海 AI 公司 / 外资 AI 中国分部

候选：做出海 AI 产品的中国公司（北美/东南亚业务）。HuggingFace / Cohere / Together.ai / Replicate 在国内**没有**分部。

**真实薪资**：中国本地档 25-45K × 14，但工时美国时区。

## 评估流程（每个 JD 进 Block A 之前必须执行）

1. **真实门槛硬筛**：JD 命中任何 Tier D 触发关键词 → SKIP，不进 A-F
2. **公司归类**：对照本文档 Tier A/B/C 公司清单 + WebSearch 该公司画像
3. **画像反推**：评估"这家公司会不会真的给候选人面试机会"，不是"JD 写的能力候选人有没有"
4. **写入 report 头**：
   ```markdown
   **Tier:** A1 / A2 / B1 / B2 / B3 / C1 / C2 / D / 未明确
   ```
5. Tier A/B → 进 Block A-F 全评估
6. Tier C → 8 月底前给"延后投递"建议
7. Tier D → SKIP，写一句"不在真实可达池"理由

## 元规则（写给未来的 Claude）

之前 v1/v2 推错的根本原因：把"市场需求"和"个人可达"混淆了。市场上 AI 工程师需求大是真的，但具体到顶尖公司他们对"AI 工程师"的定义和门槛和银行/中科金财的定义完全不在一个量级。

**正确方法**：从候选人真实简历画像反推 → 哪些公司真的会给面试 → 那些公司里哪些值得去 → 投递这些。

24 个月后画像会变（如果候选人做到：1 份银行金科或字节火山的真实工作 + LangChain 5-10 个 merged PR + Elytra 200+ star + HuggingFace finetune 模型）→ 那时字节豆包应用层和月之暗面才是真的可达。

不是直接投 frontier，是先在可达池站稳，持续输出，让简历自己升值到能投 frontier 的档位。
